# Master Prompt for App Tracker Automation

This master prompt coordinates all automation activities for the insurance portal testing framework.

## Purpose
- Define overall testing strategy
- Coordinate between different prompt types
- Ensure consistency across generated tests
- Maintain framework standards

## Key Principles
1. Component-driven architecture
2. Reusable test components
3. Clear separation of concerns
4. Comprehensive test coverage
5. Maintainable code structure

## Integration Points
- Story Analysis → Test Case Generation
- Test Case Generation → Playwright Code Generation
- Bug Reporting → Healing Rules
- Insurance Rules → All prompts

## Quality Gates
- Code review standards
- Test coverage requirements
- Performance benchmarks
- Security validation criteria
