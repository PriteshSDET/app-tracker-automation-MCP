# Story Analysis Prompt

## Purpose
Analyze user stories and requirements to extract testable scenarios.

## Input Format
- User story description
- Acceptance criteria
- Business requirements
- Technical specifications

## Analysis Steps
1. Identify test scenarios
2. Extract test data requirements
3. Determine test types (UI, API, Integration)
4. Identify edge cases
5. Define success criteria

## Output Format
- Test scenario list
- Data requirements
- Priority levels
- Dependencies
